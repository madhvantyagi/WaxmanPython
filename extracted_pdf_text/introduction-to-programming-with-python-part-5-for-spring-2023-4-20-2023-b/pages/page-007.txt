Example:
Write a function get_row(x,i) which takes a square matrix (=table=list of lists) and returns a list
containing the element in the ith row of matrix X.
Using a list comprehension we could write:
def get_row(x,i):
return [ x[i][j] for j in range(len(x)) ]
Or …we could also write
def get_row(x,i):
return x[i]
since the ith row of x is just the ith sublist of x.
However there is an important difference between these two implementations of get_row().
The first version creates a new list made up of the elements of row i of matrix X.
The second one, which is much faster, just returns a reference to the ith row of matrix X.
In the following example, we call both versions of get_row(m,0) on the matrix m=[[1,2],[3,4]].
Notice that both versions return what seems to be the same result: [1, 2]
Question: What is the important practical difference between the two versions?
Hint: What happens when we change list b after it is returned from get_row(). Why does this happen?
