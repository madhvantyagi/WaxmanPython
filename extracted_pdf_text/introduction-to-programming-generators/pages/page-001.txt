Generators
What is the difference between:
1. print(sum([1,2,3,4,5]))
2. print(sum(range(1,6)))
3. print(sum(i for i in range(1,6)))
since they all print 15?
Answer
The three code snippets all compute and print the sum of numbers from 1 to 5, resulting in 15, but they
differ in their implementation details, primarily around how they handle data in memory:
1. `print(sum([1,2,3,4,5]))`:
- Data Structure: This uses a list `[1, 2, 3, 4, 5]` explicitly defined with all the elements in memory at
once.
- Memory Usage: The numbers are all stored in memory simultaneously, making this approach less
memory-efficient for large data sets.
- Clarity: It's very clear and straightforward, showing exactly which numbers are being summed.
2. `print(sum(range(1,6)))`:
- Data Structure: `range(1, 6)` creates a range object that generates numbers on-the-fly from 1 to 5.
- Memory Usage: More memory-efficient, as `range` does not store all the numbers at once but
produces them one at a time as needed during the iteration.
- Clarity: Still clear and concise, and the `range` function is commonly used in Python for generating
sequences of numbers.
3. `print(sum(i for i in range(1,6)))`:
- Data Structure: This uses a generator expression `(i for i in range(1, 6))` which also generates
numbers on-the-fly.
- Memory Usage: Similarly to `range`, this method is memory-efficient because it doesn't hold all the
elements in memory simultaneously but yields them one by one.
- Flexibility: Offers more flexibility than `range` alone, as you can easily add more complex
conditions or transformations to each element before they are summed.
