1. Dictionaries: The Core Model pp. 1-4

The PDF introduces a dictionary by comparing it to an online dictionary or phone book: you use a key to look up a value. In Python, a dictionary maps keys to values.

Behind the scenes: A Python dict is a hash table. Python computes a hash from the key, uses that hash to find a storage location, and then retrieves the associated value. This is why lookups like prices['IBM'] are fast on average, and why keys must be hashable.

Basic Creation and Access pp. 2-4

s = {
    'name': 'GOOG',
    'shares': 100,
    'price': 490.10
}

name = s['name']
cost = s['shares'] * s['price']

s['shares'] = 75
s['date'] = '2007-06-07'
Operation Meaning Exam trap
{} or dict() Create an empty dictionary. {} is an empty dict, not an empty set. Empty set is set().
len(d) Return the number of key-value pairs. Counts dictionary items, not total nested values.
d[key] Return the value for key. Raises KeyError if missing.
d[key] = value Add or overwrite a key-value pair. Same syntax for insert and update.
del d[key] Remove a key-value pair. Raises KeyError if missing.
key in d Tests whether key exists. It checks keys, not values.
key not in d Same as not key in d. Still checks keys only.
d.get(key, default) Return value if present, otherwise default. Does not insert the key.
d.pop(key[, default]) Remove and return value. Without a default, missing key raises KeyError.
d.clear() Remove all items. The same dictionary object becomes empty.
dict.fromkeys(seq, value) Create a new dict with keys from seq. Every key receives the same value object; dangerous if that value is mutable.
d.copy() Return a shallow copy. Nested mutable objects are still shared.

Membership and Default Lookup p. 4

prices = {
    'GOOG': 490.1,
    'AAPL': 123.5,
    'IBM': 91.5,
    'MSFT': 52.13
}

if 'IBM' in prices:
    p = prices['IBM']
else:
    p = 0.0

p = prices.get('IBM', 0.0)

The two examples produce the same value for existing and missing keys. The difference is style and concision: get packages the membership test and retrieval into one expression.

What Can Be a Key? p. 4

Strings are common keys, but numbers and tuples can also be keys. Lists, sets, and dictionaries cannot be keys because they are mutable and unhashable.

prices = {}
prices[('IBM', '2015-02-03')] = 91.23
prices['IBM', '2015-02-04'] = 91.42  # same kind of tuple key
Trap: In prices['IBM', '2015-02-04'], the comma creates a tuple. The key is ('IBM', '2015-02-04'), not two separate indexes.

Likely Test Questions

  • Does 'IBM' in prices check keys or values? It checks keys.
  • What happens if prices['TSLA'] is used and 'TSLA' is missing? KeyError.
  • What does prices.get('TSLA', 0.0) return if missing? 0.0.
  • Can [1, 2] be a dictionary key? No, lists are mutable and unhashable.
  • Can (1, 2) be a dictionary key? Yes, because a tuple of hashable values is hashable.

2. Tabulation and Counter pp. 5-9

The PDF uses dictionaries to solve a tabulation problem: count the total number of shares for each stock symbol.

Manual Dictionary Tabulation p. 5

portfolio = [
    ('ACME', 50, 92.34),
    ('IBM', 75, 102.25),
    ('PHP', 40, 74.50),
    ('IBM', 50, 124.75)
]

total_shares = {s[0]: 0 for s in portfolio}
for name, shares, _ in portfolio:
    total_shares[name] += shares

# {'ACME': 50, 'IBM': 125, 'PHP': 40}
Behind the scenes: The dictionary comprehension runs first. It creates one key for each distinct stock symbol and sets every value to 0. If a symbol appears twice, like 'IBM', the later assignment still sets the same key to 0. Then the loop adds shares to the existing count.

The underscore in for name, shares, _ in portfolio is just a normal variable name used by convention to mean "I am ignoring this value." It receives the third tuple item, the price, but the program does not use it.

Counter Version pp. 5-9

from collections import Counter

total_shares = Counter()
for name, shares, _ in portfolio:
    total_shares[name] += shares

# Counter({'IBM': 125, 'ACME': 50, 'PHP': 40})

Counter is a subclass of dict specialized for counting hashable objects. Elements are dictionary keys; counts are dictionary values.

Counter behavior Example Meaning
Create from iterable Counter("banana") Counts characters: a:3, n:2, b:1.
Missing key Counter("apple")['z'] Returns 0, not KeyError.
update c.update(['red', 'blue']) Adds counts from iterable.
subtract c.subtract(['red']) Subtracts counts and can create zero or negative counts.
Cleanup c = +c or c += Counter() Removes zero and negative counts.
most_common(n) word_counts.most_common(3) Returns top n elements with counts.
elements() list(c.elements()) Repeats each element according to its positive count.
total() word_counts.total() Python 3.10+: sum of all counts.

Counter Arithmetic p. 8

from collections import Counter

c1 = Counter(a=4, b=2)
c2 = Counter(a=1, b=3, c=1)

print(c1 + c2)  # Counter({'a': 5, 'b': 5, 'c': 1})
print(c1 - c2)  # Counter({'a': 3})
print(c1 & c2)  # Counter({'b': 2, 'a': 1})
print(c1 | c2)  # Counter({'a': 4, 'b': 3, 'c': 1})
  • + adds corresponding counts.
  • - subtracts counts and discards zero or negative results.
  • & keeps minimum positive counts, like multiset intersection.
  • | keeps maximum counts, like multiset union.
Trap: Counter.subtract can leave negative counts, but the - operator discards negative and zero results. Those are not the same behavior.

When to Use Counter and When Not To pp. 8-9

Use Counter for word counts, item frequencies, votes, inventory-style tallies, sparse integer counts, and quick frequency distributions. Avoid it when you need a custom insertion-order object, mutable objects as keys, or direct floating-point/non-integer arithmetic as the main meaning of the values.

Performance Notes p. 9

Creating a Counter from an iterable is linear in the number of input elements. Most updates and lookups are average constant time because Counter inherits dictionary hash-table behavior.

3. Dictionary Views, Lists, and Order pp. 10-13

Empty Dicts and Constructing from Pairs p. 10

prices = {}
prices = dict()

pairs = [('IBM', 125), ('ACME', 50), ('PHP', 40)]
d = dict(pairs)

{} is idiomatic for an empty dictionary. dict(pairs) is useful when you already have key-value pairs.

list(d) vs d.keys() pp. 10-12

prices = {'AAPL': 123.5, 'MSFT': 52.13, 'IBM': 91.5, 'GOOG': 490.1}

syms1 = list(prices)   # frozen list of keys at this moment
syms2 = prices.keys()  # dynamic dict_keys view
Expression Type / behavior Changes when dict changes?
list(d) A real list of keys. No. It is a snapshot.
d.keys() dict_keys view. Yes. It reflects later changes.
d.values() dict_values view. Yes. It reflects changed values and removals.
d.items() dict_items view of (key, value) pairs. Yes. It reflects changed pairs. 
dishes = {'eggs': 2, 'sausage': 1, 'bacon': 1, 'spam': 500}
keys = dishes.keys()
values = dishes.values()

del dishes['eggs']
del dishes['sausage']

print(list(keys))    # ['bacon', 'spam']
print(list(values))  # [1, 500]
Behind the scenes: A view does not copy all keys or values into a new container. It points at the dictionary and exposes its current contents. That saves memory and keeps the view live.

View Set Operations p. 13

dishes = {'bacon': 1, 'spam': 500}
keys = dishes.keys()

print(keys & {'eggs', 'bacon', 'salad'})  # {'bacon'}
print(keys ^ {'sausage', 'juice'})        # {'juice', 'sausage', 'bacon', 'spam'}

Key views support set-like operations because dictionary keys are unique and hashable. Values views do not behave like sets in the same way because values are not necessarily unique or hashable.

Dictionary Order p. 13

The PDF states that keys appear in insertion order in modern Python. Converting a dict to a list preserves that key order.

od = collections.OrderedDict()
od['apple'] = 1
od['banana'] = 2
od['orange'] = 3
od['grape'] = 4

for key, value in od.items():
    print(key, value)

OrderedDict is a dictionary subclass designed to guarantee order. In current Python versions, normal dicts preserve insertion order too, but OrderedDict still signals that order is part of the design.

Trap: Iterating over a dictionary means iterating over keys, not values and not (key, value) pairs. Use d.items() when you need both.

4. Comprehensions, Shallow Copy, Deep Copy pp. 14-17

Dictionary Comprehensions p. 14

results = {n: n ** 2 for n in range(10)}
# {0: 0, 1: 1, 2: 4, ..., 9: 81}

A dictionary comprehension processes an iterable and creates key-value pairs. In this example, each key is n and each value is n ** 2.

PDF prime-number problem p. 14

The PDF asks for a dictionary mapping n to the nth prime for the first k primes, with k provided by the user. The comprehension should use an already-computed list of primes.

def is_prime(x):
    if x < 2:
        return False
    for d in range(2, int(x ** 0.5) + 1):
        if x % d == 0:
            return False
    return True

def first_k_primes(k):
    primes = []
    n = 2
    while len(primes) < k:
        if is_prime(n):
            primes.append(n)
        n += 1
    return primes

k = 5
primes = first_k_primes(k)
prime_dict = {i + 1: primes[i] for i in range(k)}
# {1: 2, 2: 3, 3: 5, 4: 7, 5: 11}
Likely professor angle: The dictionary comprehension is not where the primality algorithm has to live. The PDF says k is provided and is not part of the comprehension. Expect a question asking what the comprehension alone produces once the prime list exists.

Shallow Copy p. 14

original_dict = {'a': 1, 'b': 2, 'c': 3}
copy_dict = original_dict.copy()

A shallow copy creates a new outer dictionary, but it does not recursively copy nested mutable objects.

Deep Copy pp. 16-17

import copy

my_dict = {'a': [1, 2], 'b': [3, 4]}
my_dict_copy = copy.deepcopy(my_dict)

my_dict_copy['a'][0] = 5

print(my_dict)       # {'a': [1, 2], 'b': [3, 4]}
print(my_dict_copy)  # {'a': [5, 2], 'b': [3, 4]}
Behind the scenes: Assignment makes another name for the same object. Shallow copy makes a new outer dict but reuses nested list/dict objects. Deep copy recursively creates independent nested objects.
Operation Outer dict copied? Nested mutable values copied?
b = a No No
b = a.copy() Yes No
b = dict(a) Yes No
b = copy.deepcopy(a) Yes Yes, recursively

my_func(apple=3, banana=5, orange=2)
apple: 3 banana: 5 orange: 2

The PDF Trace Program pp. 18-19

def z(a, *b, **c):
    print(type(a), type(b), type(c))
    print(a, b[1], c, sep='\n')
    print(len(c), list(c.items()))
    g = list(c.items())
    c[g[0]] = 156
    print(c.items())
    print(type(c.items()))
    d = [i for i in c.keys()]
    print(c[d[0]])
    print()
    z = list(c)
    print(c[list(c)[0]])
    print()
    for i in iter(c):
        print(i)

z(12, 3, 4, 5, k=87, l=19)

Step-by-step state

Variable Value after call Why
a 12 First positional argument.
b (3, 4, 5) Extra positional arguments become a tuple.
c {'k': 87, 'l': 19} Keyword arguments become a dict.

Then g = list(c.items()) creates [('k', 87), ('l', 19)]. The line c[g[0]] = 156 uses ('k', 87) as a tuple key. Tuples can be keys when their elements are hashable.

<class 'int'> <class 'tuple'> <class 'dict'> 12 4 {'k': 87, 'l': 19} 2 [('k', 87), ('l', 19)] dict_items([('k', 87), ('l', 19), (('k', 87), 156)]) <class 'dict_items'> 87 87 k l ('k', 87)
Trap: The local assignment z = list(c) inside the function reuses the name z, but it does not matter after the function call. Inside the function, after that assignment, z refers to a list, not the function. This would matter only if the function tried to call z(...) again after rebinding the name.

6. Scrabble Descrambler and Pickle pp. 20-22

Signature Function p. 20

The signature of a word is its letters sorted into lexicographic order.

def signature(n):
    return ''.join(sorted(n))

print(signature('stop'))  # opst

sorted(n) returns a list of sorted characters. ''.join(...) stitches those characters into a string.

Dictionary Design for the Descrambler pp. 20-22

The Scrabble program stores all six-letter legal words in a dictionary:

  • Key: the sorted signature, such as 'ACENRT'.
  • Value: a list of all words with that same signature.
d = {}
for w in words:
    sig = signature(w)
    if sig not in d:
        d[sig] = []
    d[sig].append(w)

If the user enters letters with the same signature, the program looks up that signature and prints all matching words.

Behind the scenes: This is a reverse lookup table. Instead of searching every word each time the user enters tiles, the program pays the cost once to build a dictionary. Later lookups are fast because the signature directly indexes the list of matching words.

Pickle pp. 21-22

pickle serializes Python objects into bytes and deserializes bytes back into Python objects. The PDF uses it to save and reload the Scrabble dictionary.

import pickle

numbers = [1, 2, 3, 4, 5]
with open("numbers.pkl", "wb") as f:
    pickle.dump(numbers, f)

with open("numbers.pkl", "rb") as f:
    restored_numbers = pickle.load(f)

print(restored_numbers)
Trap: Pickle files are binary. Use "wb" to write and "rb" to read. Also, do not load pickle data from an untrusted source; unpickling can execute code. The PDF calls pickle powerful, but exam answers should remember this security issue.

Control Flow in the PDF Program pp. 21-22

  • Ask whether to create or load: mode=input("Create or Load C or L: ").
  • If mode.upper() == 'C', read the word file and build the dictionary.
  • Otherwise, open the pickle file and load the dictionary.
  • Loop until the user enters done.
  • If input length is not 6, print an error.
  • Otherwise uppercase, sort into signature, and look up in d.
  • At the end, write the dictionary back to the pickle file.

7. Inverted Index, Squish, Groupby, Defaultdict pp. 23-28

Inverted Index Problem p. 23

An inverted index maps each word to where it appears. The PDF starts with a simplified version:

invertedDict["cat"] = [4, 4, 4, 6, 6, 8, 8, 8, 8]

This means cat appears 3 times on line 4, 2 times on line 6, and 4 times on line 8.

Squish pp. 23-24

squish compresses repeated consecutive line numbers into (line, count) pairs.

def squish(x):
    if not x:
        return []
    result = []
    count = 1
    current = x[0]
    for i in range(1, len(x)):
        if x[i] == current:
            count += 1
        else:
            result.append((current, count))
            current = x[i]
            count = 1
    result.append((current, count))
    return result

print(squish([4, 4, 4, 6, 6, 8, 8, 8, 8]))
# [(4, 3), (6, 2), (8, 4)]
Trap: This version only works as intended when equal values are consecutive. If the list is [4, 6, 4], it returns [(4, 1), (6, 1), (4, 1)], not [(4, 2), (6, 1)].

Groupby Version pp. 24-25

from itertools import groupby

def squish(x):
    return [(key, len(list(group))) for key, group in groupby(x)]

groupby returns an iterator of (key, group) pairs. The group is itself an iterator, not a list. That is why the code uses list(group) before calling len.

from itertools import groupby

data = [1, 1, 2, 2, 2, 3, 3, 3, 3]
for key, group in groupby(data):
    print(f"Key: {key}, Group: {list(group)}")
Key: 1, Group: [1, 1] Key: 2, Group: [2, 2, 2] Key: 3, Group: [3, 3, 3, 3]
Behind the scenes: groupby streams through the iterable. It only groups adjacent items with the same key. It does not search the whole iterable for all matching values unless the iterable has already been sorted by the same grouping key.

Original Inverted Index Shape p. 26

The original program removes punctuation, loops over lines, splits words, and builds a dictionary where each word maps to a count and a list of line numbers.

def clean(x):
    s = {'.', ',', '-'}
    return ''.join(i for i in x if i not in s)

d = {}
ln = 0
for line in open('GB.txt'):
    ln += 1
    line = clean(line)
    words = line.split()
    for word in words:
        if word not in d:
            d[word] = []
            d[word].append(1)
            d[word].append([ln])
        else:
            d[word][0] += 1
            d[word][1].append(ln)

For each word, the first stored value is total count and the second stored value is a list of line numbers. A cleaner design is to use a dictionary like {'count': 0, 'lines': []} for each word.

Refactored Version with Defaultdict pp. 27-28

from collections import defaultdict

def build_inverted_index(file_path):
    inverted_index = defaultdict(lambda: {'count': 0, 'lines': []})
    with open(file_path, 'r') as file:
        for ln, line in enumerate(file, 1):
            cleaned_line = clean(line)
            words = cleaned_line.split()
            for word in words:
                entry = inverted_index[word]
                entry['count'] += 1
                entry['lines'].append(ln)

    for word, data in inverted_index.items():
        data['lines'] = squish(sorted(data['lines']))
    return inverted_index

defaultdict automatically creates a default value for a missing key. In this case, each new word starts with {'count': 0, 'lines': []}.

Sentinel Value in Squish p. 28

def squish(x):
    if not x:
        return []
    result = []
    current = x[0]
    count = 1
    for number in x[1:] + [None]:
        if number == current:
            count += 1
        else:
            result.append((current, count))
            current = number
            count = 1
    return result

The appended None forces one final mismatch after the real data ends. That makes the loop append the last group without needing duplicate cleanup code after the loop.

Trap: x[1:] + [None] creates a new list. It works because x is a list. If x were a tuple or iterator, this exact expression would need adjustment.

8. CSV Grade Intervals and Bar Chart pp. 29-34

Generating a CSV of Scores p. 29

The PDF creates fake student names and exam scores drawn from a normal distribution with mean 75 and standard deviation 15.

import os
import csv
import numpy as np
from faker import Faker

fake = Faker()

def generate_data_normal_dist(num_rows, mean=75, std_dev=15):
    data = []
    for _ in range(num_rows):
        last_name = fake.last_name()
        first_name = fake.first_name()
        score = np.random.normal(mean, std_dev)
        score = min(max(score, 0), 100)
        data.append([last_name, first_name, score])
    return data
  • np.random.normal(mean, std_dev) produces scores centered around 75.
  • min(max(score, 0), 100) clamps values below 0 up to 0 and values above 100 down to 100.
  • csv.writer writes rows to the file.
  • newline='' prevents extra blank lines on some systems when writing CSV files.
  • Faker generates realistic fake names for test data; the PDF notes it can be installed with pip install Faker.

Reading CSV and Filtering Grades p. 31

with open(csv_file_path, mode="r") as file:
    csv_reader = csv.reader(file)
    next(csv_reader)
    grades = []
    for row in csv_reader:
        grades.append(row[2])

positive_float_grades = [
    float(grade)
    for grade in grades
    if grade and float(grade) > 0
]
Behind the scenes: CSV fields are read as strings. Even if the file visually contains 87.5, Python reads it as '87.5'. The list comprehension converts nonblank positive grades to floats.
Trap: The PDF's missing-file branch prints csv_file_name, but the path variable shown is csv_file_path. In real code that would cause a NameError if the file does not exist.

Counting Grade Intervals p. 31

def count_numbers_in_intervals(positive_float_grades):
    interval_counts = {}
    for interval in range(1, 101, 10):
        count = sum(
            1 for num in positive_float_grades
            if interval <= num < interval + 10
        )
        interval_counts[f"{interval}-{interval + 9}"] = count
    return interval_counts

This creates labels 1-10, 11-20, ..., 91-100 if the label uses interval + 9. The condition uses a half-open range: lower bound included, upper bound excluded.

Important interval trap: For interval = 1, the condition is 1 <= num < 11. That includes 10.0. For interval = 91, the condition is 91 <= num < 101. That includes 100.0. The label is 91-100.

Matplotlib Bar Chart pp. 32-33

import matplotlib.pyplot as plt

def plot_bar_graph(data_dict):
    keys = data_dict.keys()
    values = data_dict.values()

    plt.figure(figsize=(8, 4))
    plt.bar(keys, values)
    plt.xlabel('Grade intervals')
    plt.ylabel('Number of grades')
    plt.title('Midterm grade distribution')
    plt.show()

pyplot gives a MATLAB-like plotting interface. The dictionary keys become x-axis labels and the dictionary values become bar heights.

Line What it does Behind-the-scenes behavior
plt.figure(figsize=(8, 4)) Creates a new figure 8 inches wide and 4 inches tall. At default 100 DPI, that is about 800 by 400 pixels.
plt.bar(keys, values) Draws a bar for each interval. The order follows dictionary insertion order, so the intervals appear as inserted.
plt.xlabel, plt.ylabel, plt.title Add text labels. They attach to the current active axes.
plt.show() Displays the chart window or inline output depending on environment. In scripts, it can block execution until the chart is closed.

Groupby Interval Version pp. 33-34

from itertools import groupby

def count_numbers_in_intervals(grades):
    interval_counts = {f"{i}-{i + 9}": 0 for i in range(1, 101, 10)}

    def interval_key(grade):
        lower = int((grade - 1) // 10) * 10 + 1
        upper = int((grade - 1) // 10) * 10 + 10
        return f"{lower}-{upper}"

    for key, group in groupby(sorted(grades), interval_key):
        interval_counts[key] = len(list(group))
    return interval_counts

The purpose is to sort grades, group them by the interval key, then count each group. Sorting is required because groupby only groups consecutive items.

Subtle source-specific trap: The PDF initializes labels with f"{i}-{i + 9}", but the shown interval_key returns an upper bound of ... + 10. That means the prefilled label 1-10 matches, but the formula for a grade of 100 returns 91-100; for ordinary bins it is consistent with labels like 31-40. Be careful not to mix labels like 1-9 or 1-11.

9. Professor-Style Print and Reasoning Questions

Question 1: Dict lookup vs get

prices = {'IBM': 91.5, 'MSFT': 52.13}
print(prices.get('GOOG', 0.0))
print(prices['GOOG'])
Answer:
0.0 KeyError: 'GOOG'

get returns the default for a missing key. Bracket lookup requires the key to exist.

Question 2: Tuple key hidden by commas

d = {}
d['IBM', '2015-02-04'] = 91.42
print(list(d.keys()))
print(d[('IBM', '2015-02-04')])
Answer:
[('IBM', '2015-02-04')] 91.42

The comma creates a tuple key. Parentheses are optional in this expression.

Question 3: Dynamic view vs frozen list

d = {'a': 1, 'b': 2}
k1 = d.keys()
k2 = list(d)
d['c'] = 3
del d['a']
print(list(k1))
print(k2)
Answer:
['b', 'c'] ['a', 'b']

k1 is a live view. k2 is a list snapshot created before the mutation.

Question 4: Counter missing keys and cleanup

from collections import Counter
c = Counter("apple")
print(c['z'])
c.subtract(['p', 'p', 'p', 'z'])
print(c)
print(+c)
Answer:
0 Counter({'a': 1, 'l': 1, 'e': 1, 'p': -1, 'z': -1}) Counter({'a': 1, 'l': 1, 'e': 1})

Counter returns 0 for missing keys. subtract allows negative counts. Unary plus removes zero and negative counts.

Question 5: Counter arithmetic

from collections import Counter
c1 = Counter(a=2, b=1)
c2 = Counter(a=5, b=1, c=3)
print(c1 - c2)
print(c1 & c2)
print(c1 | c2)
Answer:
Counter() Counter({'a': 2, 'b': 1}) Counter({'a': 5, 'c': 3, 'b': 1})

c1 - c2 discards nonpositive results. Intersection takes minimum counts. Union takes maximum counts.

Question 6: Shallow copy trap

a = {'x': [1, 2], 'y': 3}
b = a.copy()
b['x'][0] = 99
b['y'] = 4
print(a)
print(b)
Answer:
{'x': [99, 2], 'y': 3} {'x': [99, 2], 'y': 4}

The outer dictionaries are separate, so changing b['y'] does not affect a['y']. But both dictionaries share the same nested list.

Question 7: Deep copy contrast

import copy
a = {'x': [1, 2]}
b = copy.deepcopy(a)
b['x'].append(3)
print(a)
print(b)
Answer:
{'x': [1, 2]} {'x': [1, 2, 3]}

Deep copy recursively copies the nested list, so appending to b['x'] does not mutate a['x'].

Question 8: Args and kwargs

def f(a, *b, **c):
    print(a)
    print(b)
    print(c)
    print(type(b), type(c))

f(10, 20, 30, x=1, y=2)
Answer:
10 (20, 30) {'x': 1, 'y': 2} <class 'tuple'> <class 'dict'>

*b gathers extra positional arguments into a tuple. **c gathers keyword arguments into a dictionary.

Question 9: Iterating over a dictionary

d = {'first': 1, 'second': 2}
for x in d:
    print(x)
for x in d.items():
    print(x)
Answer:
first second ('first', 1) ('second', 2)

Plain dictionary iteration yields keys. items() yields (key, value) tuples.

Question 10: Signature and anagram dictionary

def signature(n):
    return ''.join(sorted(n))

words = ['stop', 'pots', 'tops', 'post', 'spot', 'cat']
d = {}
for w in words:
    sig = signature(w)
    if sig not in d:
        d[sig] = []
    d[sig].append(w)

print(signature('opts'))
print(d[signature('opts')])
Answer:
opst ['stop', 'pots', 'tops', 'post', 'spot']

Every word with letters o, p, s, t has the same signature opst.

Question 11: Squish with nonconsecutive repeats

from itertools import groupby
def squish(x):
    return [(key, len(list(group))) for key, group in groupby(x)]

print(squish([4, 4, 6, 4, 4]))
Answer:
[(4, 2), (6, 1), (4, 2)]

groupby groups consecutive equal values only. The two groups of 4 are separated by 6, so they stay separate.

Question 12: Groupby must be sorted by key

from itertools import groupby
grades = [95, 12, 18, 91, 17]

def interval_key(grade):
    return f"{int((grade - 1) // 10) * 10 + 1}-{int((grade - 1) // 10) * 10 + 10}"

for key, group in groupby(grades, interval_key):
    print(key, list(group))
Answer:
91-100 [95] 11-20 [12, 18] 91-100 [91] 11-20 [17]

The same interval appears twice because the list is not sorted by interval. The PDF's grade-count version uses groupby(sorted(grades), interval_key) to avoid this.

Question 13: Interval boundary prediction

grades = [0, 1, 10, 10.5, 11, 100]
positive = [g for g in grades if g and g > 0]
for interval in range(1, 101, 10):
    count = sum(1 for num in positive if interval <= num < interval + 10)
    if count:
        print(f"{interval}-{interval + 9}: {count}")
Answer:
1-10: 3 11-20: 1 91-100: 1

0 is removed by the positive filter. 1, 10, and 10.5 all satisfy 1 <= num < 11. 11 starts the next bin. 100 belongs to 91-100.

Question 14: Defaultdict missing key

from collections import defaultdict
d = defaultdict(lambda: {'count': 0, 'lines': []})
d['cat']['count'] += 1
d['cat']['lines'].append(4)
print(d['cat'])
print(d['dog'])
Answer:
{'count': 1, 'lines': [4]} {'count': 0, 'lines': []}

Accessing d['dog'] creates the default dictionary for that key. This is helpful but can also create keys accidentally.

Question 15: Matplotlib data source

interval_counts = {'1-10': 2, '11-20': 5, '21-30': 0}
keys = interval_counts.keys()
values = interval_counts.values()
print(list(keys))
print(list(values))
Answer:
['1-10', '11-20', '21-30'] [2, 5, 0]

Those keys and values are what the PDF passes to plt.bar(keys, values). The labels are the x positions; the counts are bar heights.

10. Common Traps From This PDF

Dictionary traps

  • {} is an empty dict, not an empty set.
  • in checks keys, not values.
  • d[key] raises KeyError for a missing key.
  • d.get(key, default) does not insert the key.
  • Plain iteration over d yields keys.
  • d.items() is a dynamic view, not a list.

Hashability traps

  • Tuple keys work only if their contents are hashable.
  • Lists, sets, and dicts cannot be keys.
  • d['A', 'B'] uses tuple key ('A', 'B').

Counter traps

  • Missing Counter keys return 0.
  • subtract can produce negative counts.
  • c1 - c2 discards nonpositive counts.
  • elements() ignores zero and negative counts.

Copy traps

  • Assignment is not copying.
  • copy() is shallow.
  • Nested lists remain shared after shallow copy.
  • deepcopy is needed for independent nested mutable objects.

Groupby traps

  • groupby groups consecutive elements only.
  • Sort by the same key before grouping if you need all equal-key items together.
  • The group object is an iterator; once consumed, it is exhausted.
  • list(group) is used to count or inspect the group.

CSV and chart traps

  • CSV values are strings until converted.
  • next(csv_reader) skips the header row.
  • The interval test is half-open: low <= num < high.
  • plt.figure(figsize=(8, 4)) uses inches, not pixels.
  • plt.bar(keys, values) relies on matching key/value order.
