1. Sets and Unique Random Values PDF pp. 1-2, printed 110-111

Source problem pattern

The PDF starts with two related problems:

  • Ask for integers n and m with m > n. Create a list x of size n filled with random integers from 1 through m.
  • Modify the solution so all values in x are unique.

Set facts you must know

A set is a container like a list, but it can contain only one copy of any element. Sets are useful for duplicate checking because membership testing with value in s is the natural operation.

OperationMeaningTrap
s = set()Create an empty set.{} creates an empty dictionary, not an empty set.
s = set(iterable)Build a set from another iterable, removing duplicates.Order is not the point of a set.
s.add(elem)Add one element.Adding a duplicate changes nothing.
s.remove(elem)Remove an element.Raises KeyError if missing.
s.discard(elem)Remove if present.No error if missing.
s.pop()Remove and return an arbitrary element.Not first or last; arbitrary. Raises KeyError if empty.
s.clear()Remove all elements.The set object remains; it is just empty.
len(s)Number of unique elements.This is cardinality, not max index.

Implementation pattern: random list with repeats allowed

import random

n = int(input("n: "))
m = int(input("m: "))
x = []
for i in range(n):
    x.append(random.randint(1, m))  # includes both endpoints
print(x)

Implementation pattern: random list with unique values

import random

n = int(input("n: "))
m = int(input("m: "))
seen = set()
x = []
while len(x) < n:
    value = random.randint(1, m)
    if value not in seen:
        seen.add(value)
        x.append(value)
print(x)
Behind the scenes: x is a name bound to a list object. seen is a name bound to a set object. x.append(value) mutates the existing list object; it does not create a new list. seen.add(value) mutates the existing set object.
Common trap: If m < n, it is impossible to choose n unique integers from 1..m. If a loop tries anyway, it may run forever. The PDF says m > n for the first version, but for uniqueness you need at least m >= n.

Likely exam questions

  • Explain why a set removes duplicates from a list.
  • Write code that uses a set to prevent duplicate random values.
  • State the difference between remove and discard.
  • Predict whether a loop can terminate when the range contains fewer unique values than requested.

2. 2D Tables, Rows, Columns, and Diagonals PDF pp. 3-5, printed 112-114

Source problem pattern

The PDF moves from a 1D list to a two-dimensional table, represented as a list of lists. It asks for an n * m table of unique random integers from 1 through k, where k > m*n.

Creating a 2D table correctly

import random

n = int(input("rows: "))
m = int(input("cols: "))
k = int(input("max value: "))

seen = set()
x = []
for row in range(n):
    new_row = []
    for col in range(m):
        value = random.randint(1, k)
        while value in seen:
            value = random.randint(1, k)
        seen.add(value)
        new_row.append(value)
    x.append(new_row)

print(x)
Major nested-list trap: Do not create a table with a = n * [m * [0]]. That repeats references to the same inner list. Mutating one row mutates every row. The PDF later uses the safe form a = [5 * [0] for i in range(5)] for robot paths.

Rows and columns

def get_row_copy(x, i):
    return [x[i][j] for j in range(len(x[i]))]

def get_row_reference(x, i):
    return x[i]

def get_col(x, i):
    return [x[j][i] for j in range(len(x))]

For a square matrix, len(x) is both the number of rows and the number of columns. For a rectangular table, columns should use len(x) for rows and rows should use len(x[i]) for columns.

Main diagonal and anti-diagonal

The main diagonal goes upper-left to lower-right: x[0][0], x[1][1], x[2][2], and so on.

def main_diag_sum(x):
    return sum(x[i][i] for i in range(len(x)))

The other diagonal goes upper-right to lower-left: x[0][n-1], x[1][n-2], x[2][n-3], and so on.

def anti_diag_sum(x):
    n = len(x)
    return sum(x[i][n - 1 - i] for i in range(n))
Index mechanics: If n = len(x), then row i on the anti-diagonal pairs with column n - 1 - i. When i = 0, this is the last column. When i = n - 1, this is column 0.

Likely exam questions

  • Write get_col(x, i) and explain why it cannot just return x[i].
  • Trace the values visited by x[i][len(x)-1-i].
  • Explain the reference difference between returning x[i] and returning a newly built row list.

3. List Comprehensions PDF pp. 5-8, printed 114-118

General form

[expression using a value from an iterable for value in iterable if condition]

The expression runs for each value produced by the iterable that passes the optional if condition. The result is always a new list.

Squares of even numbers from 1 through 100

a = [x * x for x in range(1, 101) if x % 2 == 0]

Equivalent loop:

a = []
for x in range(1, 101):
    if x % 2 == 0:
        a.append(x * x)

Prime list example

y = [k for k in range(2, 121) if prime(k)]

This assumes prime(k) returns True for primes and False otherwise. The function call is evaluated once for each k from 2 through 120.

Dot product example

x = [1, 2, 3]
y = [10, 11, 12]
dot = sum([x[i] * y[i] for i in range(len(x))])
# 1*10 + 2*11 + 3*12 = 68

Even values with positions

z = [13, 14, 7, 12, 11, 3, 18, 20, 5, 7,
     19, 17, 10, 16, 14, 12, 5, 19, 15, 1]
k = [(i, z[i]) for i in range(20) if z[i] % 2 == 0]
# [(1, 14), (3, 12), (6, 18), (7, 20),
#  (12, 10), (13, 16), (14, 14), (15, 12)]
Behind the scenes: The comprehension creates a new list object. Each tuple (i, z[i]) is a new tuple object. The original z list is not mutated.

Row copy versus row reference

The PDF emphasizes that two versions of get_row can print the same result but behave differently after mutation.

def get_row_a(x, i):
    return [x[i][j] for j in range(len(x))]

def get_row_b(x, i):
    return x[i]

m = [[1, 2], [3, 4]]
a = get_row_a(m, 0)
b = get_row_b(m, 0)
a[0] = 99
b[1] = 88
print(m)
Output: [[1, 88], [3, 4]]. a points to a new list, so changing a[0] does not change m. b points to the exact same inner list as m[0], so changing b[1] changes the matrix.

Column sum and diagonal difference with comprehensions

def sum_col(x, i):
    return sum([x[j][i] for j in range(len(x))])

def diag_diff(x):
    n = len(x)
    d1 = sum([x[i][i] for i in range(n)])
    d2 = sum([x[i][n - 1 - i] for i in range(n)])
    return d1 - d2

make_int: representing an integer as a list of digits

The PDF notes that early Python distinguished int and long, and says modern Python 3 integers are represented like long integers internally. The class exercise represents a big integer as a list of digits.

def make_int(x: str) -> list:
    return [int(ch) for ch in x]
Trap: [int(x)] is not the same as [int(ch) for ch in x]. For "123", the first gives [123]; the second gives [1, 2, 3].

4. Big-Integer Addition and Matrix Algorithms PDF pp. 9-10, printed 119 plus later pp. 21-23, printed 129-131

Adding integer-lists digit by digit

The source asks for int_add(x, y), where integers are represented as lists of digits. Different lengths are allowed, and a final carry may add one extra digit.

def int_add(x, y):
    x = x[:]
    y = y[:]

    while len(x) < len(y):
        x.insert(0, 0)
    while len(y) < len(x):
        y.insert(0, 0)

    carry = 0
    ans = []
    for i in range(len(x) - 1, -1, -1):
        total = x[i] + y[i] + carry
        ans.insert(0, total % 10)
        carry = total // 10

    if carry:
        ans.insert(0, carry)
    return ans
Call stack and binding note: Inside int_add, the names x and y initially refer to the caller's list objects. The assignments x = x[:] and y = y[:] rebind those local names to copies, so padding with zeros does not mutate the caller's lists.

Matrix addition

def matrix_add(x, y):
    n = len(x)
    return [[x[i][j] + y[i][j] for j in range(n)] for i in range(n)]

Matrix multiplication, standard version

def matrix_mult(x, y):
    n = len(x)
    ans = []
    for i in range(n):
        row = []
        for j in range(n):
            total = 0
            for k in range(n):
                total += x[i][k] * y[k][j]
            row.append(total)
        ans.append(row)
    return ans

Matrix multiplication, Pythonic zip version

def matrix_mult(A, B):
    dot = lambda r, c: sum([r[i] * c[i] for i in range(len(r))])
    return [[dot(ri, cj) for cj in zip(*B)] for ri in A]

zip(*B) turns columns of B into tuples, so each cj acts like a column vector. The outer comprehension chooses each row ri from A.

Saddle point

A saddle point is an entry that is the minimum in its row and the maximum in its column. The PDF asks to return (value, x pos, y pos) if one exists, or "not found".

def saddle_point(x):
    n = len(x)
    for i in range(n):
        row_min = min(x[i])
        for j in range(n):
            if x[i][j] == row_min:
                col = [x[r][j] for r in range(n)]
                if x[i][j] == max(col):
                    return (x[i][j], i, j)
    return "not found"
Trap: If a row has repeated minimum values, you must test each matching position, because only one of those columns may make the value a column maximum.

5. Robot Paths: Dynamic Programming and Recursion PDF p. 11, printed 11/111

Problem

A robot starts at the upper-left square of a 5 by 5 grid and must reach the lower-right square. At each square it may move only one square right or one square down. Count how many paths exist.

Table solution from the PDF

a = [5 * [0] for i in range(5)]

for i in range(5):
    a[i][0] = 1
    a[0][i] = 1

for i in range(1, 5):
    for j in range(1, 5):
        a[i][j] = a[i - 1][j] + a[i][j - 1]

print(a[4][4])

Every cell stores the number of ways to reach that cell. A robot can only enter cell (i, j) from above (i-1, j) or from the left (i, j-1), so the count is the sum of those two counts. The top row and left column are all 1 because there is only one straight-line path to each of those boundary cells.

Final 5 by 5 table:
1   1   1   1   1
1   2   3   4   5
1   3   6  10  15
1   4  10  20  35
1   5  15  35  70

Recursive solution from the PDF

def num_paths(n, i, j):
    if i == 0 or j == 0:
        return 1
    return num_paths(n, i - 1, j) + num_paths(n, i, j - 1)

n = int(input("Please enter the 'size' of the array, n= "))
print(num_paths(n, n - 1, n - 1))
Call stack mechanics: Each recursive call gets its own local i and j. The function keeps splitting the problem until it reaches the top row or left column. Then returns flow back up the call stack and are added together.
Performance trap: The recursive version recomputes the same subproblems many times. The table version stores each answer once, so it is much faster for larger grids.

6. Base 8 Generation and Eight Queens PDF pp. 12-14, printed 120-122

Generating all eight-digit base-8 integers

Base 8 uses digits 0 through 7. The PDF shows eight nested loops, one for each digit, producing lists such as [0, 0, 0, 0, 0, 0, 0, 0] through [7, 7, 7, 7, 7, 7, 7, 7].

for i0 in range(8):
    for i1 in range(8):
        for i2 in range(8):
            for i3 in range(8):
                for i4 in range(8):
                    for i5 in range(8):
                        for i6 in range(8):
                            for i7 in range(8):
                                q = [i0, i1, i2, i3, i4, i5, i6, i7]
                                print(q)

This prints 8 ** 8 configurations because each of 8 positions has 8 choices.

get_num(n): decimal to fixed 8-digit base-8 list

def get_num(n):
    digits = [0] * 8
    for i in range(7, -1, -1):
        digits[i] = n % 8
        n = n // 8
    return digits

Example: get_num(10) returns [0, 0, 0, 0, 0, 0, 1, 2] because decimal 10 is base-8 12.

Eight Queens representation

The PDF connects base-8 configurations to the Eight Queens problem. A configuration can be represented as a vector q of length 8. Usually, column i contains one queen, and q[i] gives that queen's row.

  • Row conflict: two columns have the same row value.
  • Main diagonal conflict: two queens have equal row + col.
  • Other diagonal conflict: two queens have equal row - col.

Permutation solution from the PDF

from itertools import permutations

col = [0, 1, 2, 3, 4, 5, 6, 7]
for vec in permutations(col):
    if (8 == len(set(vec[i] + i for i in col))
           == len(set(vec[i] - i for i in col))):
        print(vec)

permutations(col) already prevents row conflicts because every row value appears exactly once. The two set lengths test diagonal conflicts. If there are 8 distinct vec[i] + i values and 8 distinct vec[i] - i values, no two queens share a diagonal.

diagonal_threat helper

def diagonal_threat(q):
    for i in range(len(q)):
        for j in range(i + 1, len(q)):
            if q[i] + i == q[j] + j:
                return True
            if q[i] - i == q[j] - j:
                return True
    return False
from itertools import permutations

candidates = permutations([0, 1, 2, 3, 4, 5, 6, 7])
count = 0
for p in candidates:
    if diagonal_threat(p):
        continue
    count += 1
    print("Solution number ", count, ": ", p)
print()
Trap: permutations returns tuples, not lists. You can index tuples the same way, but you cannot mutate them.

7. Unpacking Sequences and Iterables PDF pp. 15-18, printed 123-126

Parallel assignment

lax_coordinates = (33.9425, -118.408056)
latitude, longitude = lax_coordinates

Python asks the iterable for values and binds each target name to one value. Without a star target, the number of values must exactly match the number of names.

Swapping variables

b, a = a, b

The right side is evaluated first, forming the pair of current values. Then the left names are rebound. No temporary variable is needed.

Unpacking into function calls

divmod(20, 8)    # (2, 4)
t = (20, 8)
divmod(*t)      # same as divmod(20, 8)
quotient, remainder = divmod(*t)

The *t in a call expands the iterable into separate positional arguments.

Ignoring values with underscore

import os
_, filename = os.path.split("/home/luciano/.ssh/id_rsa.pub")
print(filename)  # id_rsa.pub

_ is just a normal name, but by convention it means "I do not plan to use this value."

Using star to grab excess items

a, b, *rest = range(5)      # a=0, b=1, rest=[2, 3, 4]
a, b, *rest = range(3)      # a=0, b=1, rest=[2]
a, b, *rest = range(2)      # a=0, b=1, rest=[]
a, *body, c, d = range(5)   # a=0, body=[1, 2], c=3, d=4
*head, b, c, d = range(5)   # head=[0, 1], b=2, c=3, d=4
Important: In assignment, the starred target receives a list. In a function definition like def fun(..., *rest), the excess arguments are collected into a tuple.

Star unpacking in function calls and literals

def fun(a, b, c, d, *rest):
    return a, b, c, d, rest

fun(*[1, 2], 3, *range(4, 7))
# (1, 2, 3, 4, (5, 6))

*range(4), 4
# (0, 1, 2, 3, 4)

[*range(4), 4]
# [0, 1, 2, 3, 4]

{*range(4), 4, *(5, 6, 7)}
# {0, 1, 2, 3, 4, 5, 6, 7}

Nested unpacking

metro_areas = [
    ("Tokyo", "JP", 36.933, (35.689722, 139.691667)),
    ("Delhi NCR", "IN", 21.935, (28.613889, 77.208889)),
    ("Mexico City", "MX", 20.142, (19.433333, -99.133333)),
    ("New York-Newark", "US", 20.104, (40.808611, -74.020386)),
    ("Sao Paulo", "BR", 19.649, (-23.547778, -46.635833)),
]

for name, _, _, (lat, lon) in metro_areas:
    if lon <= 0:

The last field in each record is itself a tuple. The target (lat, lon) unpacks that nested tuple directly.

8. zip, Unzipping, Dictionaries, and Transpose PDF pp. 19-23, printed 127-131

Basic zip

a = [1, 2, 3]
b = ["a", "b", "c"]
print(list(zip(a, b)))
# [(1, "a"), (2, "b"), (3, "c")]

zip returns an iterator. Once you consume it, it is exhausted. Convert to list only when you need to see or reuse all pairs.

Different lengths

a = [1, 2, 3, 4]
b = ["a", "b"]
print(list(zip(a, b)))
# [(1, "a"), (2, "b")]

zip stops when the shortest iterable is exhausted.

Unzipping

pairs = [(1, "a"), (2, "b"), (3, "c")]
numbers, letters = zip(*pairs)
print(numbers)  # (1, 2, 3)
print(letters)  # ("a", "b", "c")

*pairs passes each tuple as a separate argument to zip. Then zip groups all first elements, all second elements, and so on.

Parallel iteration

for number, letter in zip([1, 2, 3], ["a", "b", "c"]):
    print(number, letter)

Creating dictionaries

keys = ["a", "b", "c"]
values = [1, 2, 3]
dictionary = dict(zip(keys, values))
print(dictionary)  # {"a": 1, "b": 2, "c": 3}

Transposing a matrix

B = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9],
]

print(list(zip(*B)))
# [(1, 4, 7), (2, 5, 8), (3, 6, 9)]

transposed = [list(group) for group in zip(*B)]
print(transposed)
# [[1, 4, 7], [2, 5, 8], [3, 6, 9]]
Mechanics of zip(*B): If B has three rows, *B passes those rows as three separate arguments: zip(B[0], B[1], B[2]). Then zip groups values by column position.

9. Strings PDF pp. 24-28, printed 132-134 and 124

Core string facts

  • Strings are immutable. You cannot change a character in place.
  • Strings are iterable, so for ch in s works.
  • Strings are sequences, so indexing and slicing work.
  • String concatenation with + creates a new string.

Reverse a string

def reverse_slicing(s):
    return s[::-1]

def reverse_loop(s):
    ans = ""
    for ch in s:
        ans = ch + ans
    return ans

def reverse_join_reversed(s):
    return "".join(reversed(s))
Behind the scenes: Because strings are immutable, ans = ch + ans makes a new string each time. reversed(s) returns an iterator over characters in reverse order; join builds the final string.

Exactly one uppercase character

def one_upper(s):
    return sum([1 for ch in s if ch.isupper()]) == 1

The PDF says the input contains only alphabetic characters and no blanks. With that assumption, isupper and islower are enough.

clear: punctuation at the end of words

def clear_loop(x):
    cleaned_words = []
    for word in x:
        if word[-1] in (".", ",", ";"):
            cleaned_word = word[:-1]
        else:
            cleaned_word = word
        cleaned_words.append(cleaned_word)
    return cleaned_words

def clear_rstrip(x):
    return [word.rstrip(".,;") for word in x]

def clear_conditional(x):
    return [word[:-1] if word[-1] in ".,;" else word for word in x]

Do these behave exactly the same?

  • clear_loop and clear_conditional remove at most one final punctuation character.
  • clear_rstrip removes all trailing characters that are in ".,;".
  • All three fail on empty words if they access word[-1]; rstrip handles empty strings safely.
x = ["hi...", "ok;", "", "a,b"]
clear_rstrip(x)       # ["hi", "ok", "", "a,b"]
# clear_loop(x) would raise IndexError on "" unless guarded.

Clearing punctuation from left, both ends, or anywhere

def clear_left(x):
    return [word.lstrip(".,;") for word in x]

def clear_both_ends(x):
    return [word.strip(".,;") for word in x]

def clear_anywhere(x):
    return ["".join(char for char in word if char not in ".,;") for word in x]

scrape(s): finding links in HTML

The PDF asks for a function that takes HTML text and returns a list of all links. A simple class-level solution is to search for href="..." patterns. This is not a full HTML parser, but it matches the string-method focus of this slide set.

def scrape(s):
    links = []
    pos = 0
    marker = 'href="'
    while True:
        start = s.find(marker, pos)
        if start == -1:
            break
        start = start + len(marker)
        end = s.find('"', start)
        if end == -1:
            break
        links.append(s[start:end])
        pos = end + 1
    return links

Mechanics: find returns the index where the marker begins. If missing, it returns -1. After finding href=", the actual URL starts after the marker and ends at the next quote.

Count the ones from 1 through n

def count_ones(n):
    total = 0
    for i in range(1, n + 1):
        total += str(i).count("1")
    return total

For n = 20, the answer is 12: one from 1, one from 10, two from 11, and one each from 12 through 19.

The with statement creates a context. When control leaves the block, the file is closed automatically. Without with, you must remember file.close().

            break
        print(chunk, end="")

read() returns an empty string at end-of-file. The walrus operator := assigns and returns the assigned value so the loop condition can test it.

Sorting by last name, then first name

The PDF shows a file bnames.txt with first name, last name, midterm score, and final score. It stores rows in a 2D list, computes the class average, sorts, and writes results.

def last_then_first(x):
    return (x[1], x[0])

cav = 0
grades = []

for line in open("bnames.txt"):
    line = line.split()
    line[2] = float(line[2])
    line[3] = float(line[3])
    grades.append(line)
    cav += (line[2] + line[3]) / 2

cav = cav / len(grades)

outfile = open("scores.txt", "w")
grades.sort(key=last_then_first)

for line in grades:
    av = (line[2] + line[3]) / 2
    out_line = (
        format(line[0], "<10s")
        + format(line[1], "<10s")
        + str(line[2]) + " "
        + str(line[3]) + " "
        + str(av) + " "
        + str(av - cav) + "\n"
    )
    outfile.write(out_line)

outfile.close()
Sorting trap: The source uses return x[1] + x[0]. A safer key is (x[1], x[0]), because tuple sorting clearly sorts by last name first, then first name. String concatenation can be ambiguous: ("Ann", "Ab") and ("Nna", "A") can collide as concatenated strings in contrived cases.

11. Professor-Style Output and Reasoning Drills

Drill 1: row copy versus reference PDF p. 7, printed 116

def get_row_a(x, i):
    return [x[i][j] for j in range(len(x))]

def get_row_b(x, i):
    return x[i]

m = [[1, 2], [3, 4]]
a = get_row_a(m, 0)
b = get_row_b(m, 0)
a.append(99)
b.append(88)
print(a)
print(b)
print(m)
Answer: 
[1, 2, 99]
[1, 2, 88]
[[1, 2, 88], [3, 4]]

a is a new list. b is an alias for m[0]. Appending to b mutates the first row of m.

Drill 2: bad 2D-list construction

a = 3 * [3 * [0]]
a[0][1] = 7
print(a)
Answer: [[0, 7, 0], [0, 7, 0], [0, 7, 0]]. The outer multiplication repeats references to the same inner list. There are three row references but only one row object.

Drill 3: safe 2D-list construction

a = [3 * [0] for i in range(3)]
a[0][1] = 7
print(a)
Answer: [[0, 7, 0], [0, 0, 0], [0, 0, 0]]. The comprehension creates a new inner list on each iteration.

Drill 4: diagonal difference

x = [
    [2, 4, 6],
    [1, 3, 5],
    [9, 8, 7],
]
print(sum([x[i][i] for i in range(len(x))]))
print(sum([x[i][len(x)-1-i] for i in range(len(x))]))
print(sum([x[i][i] for i in range(len(x))]) -
      sum([x[i][len(x)-1-i] for i in range(len(x))]))
Answer: 
12
18
-6

Main diagonal: 2 + 3 + 7 = 12. Other diagonal: 6 + 3 + 9 = 18. Difference: 12 - 18 = -6.

Drill 5: list comprehension order

x = [3, 4, 5, 6]
y = [(i, x[i] * 2) for i in range(len(x)) if x[i] % 2 == 0]
print(y)
Answer: [(1, 8), (3, 12)]. Index 1 has value 4 and index 3 has value 6. The expression stores the index and twice the value.

Drill 6: int_add carry

print(int_add([9, 9, 9], [1]))
Answer: [1, 0, 0, 0]. Pad [1] to [0, 0, 1]. Add right to left: 9+1=10, carry 1; 9+0+1=10; 9+0+1=10; final carry becomes a new leading digit.

Drill 7: robot paths small case

print(num_paths(3, 2, 2))
Answer: 6. In a 3 by 3 grid from (0,0) to (2,2), the robot must make two down moves and two right moves. The table value at (2,2) is 6.

Drill 8: star assignment

a, *b, c = range(5)
print(a)
print(b)
print(c)
Answer: 
0
[1, 2, 3]
4

The starred assignment target receives a list of the middle excess values.

Drill 9: function-call unpacking

def f(a, b, c, *rest):
    print(a, b, c, rest)

f(*[1, 2], 3, *range(4, 6))
Answer: 1 2 3 (4, 5). The call becomes f(1, 2, 3, 4, 5). The function header collects extra arguments into tuple rest.

Drill 10: zip exhaustion

z = zip([1, 2], ["a", "b"])
print(list(z))
print(list(z))
Answer: 
[(1, 'a'), (2, 'b')]
[]

zip returns an iterator. After the first list(z), it has been consumed.

Drill 11: transpose with uneven rows

B = [[1, 2, 3], [4, 5], [6, 7, 8]]
print(list(zip(*B)))
Answer: [(1, 4, 6), (2, 5, 7)]. zip stops at the shortest row, which has length 2.

Drill 12: string split traps

print("1,,2".split(","))
print(" 1  2 3 ".split())
print(" 1  2 3 ".split(" "))
Answer: 
['1', '', '2']
['1', '2', '3']
['', '1', '', '2', '3', '']

With an explicit separator, consecutive separators create empty strings. With no separator, whitespace runs are grouped and edge whitespace is ignored.

Drill 13: rstrip is not suffix removal

print("mississippi".rstrip("ip"))
print("hello...;;".rstrip(".,;"))
Answer: 
mississ
hello

rstrip(chars) removes any trailing character that belongs to the set of chars, repeatedly. It does not remove one exact suffix string.

Drill 14: strings are immutable

s = "cat"
s.upper()
print(s)
s = s.upper()
print(s)
Answer: 
cat
CAT

The first upper() returns a new string that is ignored. The assignment rebinds s to the new uppercase string.

Drill 15: file line printing

# Suppose one line read from a file is:
line = "there were\n"
print(line)
print(line, end="")
Answer: The first print outputs the line's own newline and then print's newline, so it leaves a blank-looking line after it. The second print uses the line's own newline only.

Drill 16: grades line parsing

line = "Sally 90 97 77 56 88 98\n"

scores = [int(score) for score in s[1:]]
print(s[0])
print(scores)

[90, 97, 77, 56, 88, 98]

split() removes the newline and separates on whitespace. The score strings must become integers before summing.

Drill 17: CSV header skip

    ["Name", "Age", "City"],
    ["Alice", "30", "New York"],
    ["Bob", "25", "Los Angeles"],
])

    print(row[0])

Alice
Bob

next(rows) consumes the header. The loop sees only the remaining rows.

Drill 18: eight queens diagonal sets

vec = (0, 1, 2, 3, 4, 5, 6, 7)
col = [0, 1, 2, 3, 4, 5, 6, 7]
print(len(set(vec[i] + i for i in col)))
print(len(set(vec[i] - i for i in col)))
Answer: 
8
1

The row + col values are all different: 0,2,4,...,14. The row - col values are all 0, so all queens lie on the same diagonal. This is not a valid solution.

12. Final Exam Checklist

Containers

  • Use sets for uniqueness and duplicate tests.
  • Know set operations and their errors.
  • Know when order matters and when it does not.

References

  • Returning x[i] returns a row alias.
  • Returning a comprehension returns a new list.
  • List mutation affects all aliases.

2D Lists

  • Rows: x[i].
  • Columns: [x[j][i] for j in range(len(x))].
  • Diagonals: x[i][i] and x[i][n-1-i].

Comprehensions

  • Expression first, loop second, condition last.
  • They create new lists.
  • Nested comprehensions preserve loop order.

Unpacking

  • Exact counts unless one starred target exists.
  • Starred assignment target becomes a list.
  • * in calls expands arguments.

zip

  • zip returns an iterator of tuples.
  • Stops at the shortest iterable.
  • zip(*B) transposes rows into columns.

Strings

  • Immutable: methods return new strings.
  • split() differs from split(" ").
  • rstrip(chars) treats chars as a set.

Files

  • Use with open(...) to close automatically.
  • File lines usually include \n.
  • Use csv.reader for CSV, not manual comma splitting.
