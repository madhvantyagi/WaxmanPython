1. Lambda Syntax & Rules

A lambda is a tiny anonymous function you write in a single expression.

The Golden Rules

Rule Detail
Syntax lambda arguments: expression
No return The expression is automatically returned.
One expression only No = assignments, no for/while, no if/else blocks (but ternary a if cond else b is OK).
Multiple args lambda a, b: a + b
Default args lambda a, b=10: a + b — evaluated at creation time! 
# Assigning to a variable (just like def)
square = lambda x: x ** 2
print(square(5))   # 25

# With a ternary expression (allowed!)
classify = lambda x: "even" if x % 2 == 0 else "odd"
print(classify(7))  # "odd"

# Multiple arguments
add = lambda a, b: a + b
print(add(3, 4))    # 7

2. Sorting with key=lambda

Your midterm (Q4 and Q8) always asks: "Write a function whose body is just 1 line that returns a sorted list." The answer is always sorted() + a lambda key.

🎯 3-Step Strategy

  1. Ask: What is each item in the list? (a sub-list? a tuple? a string?)
  2. Ask: What property do I sort by? (average? length? last element?)
  3. Write: sorted(data, key=lambda x: <property of x>)

Sorting a list of lists by average (Midterm Q4)

# x = one inner list like [3, 5, 7]
# property = average = sum(x) / len(x)
sorted(input_list, key=lambda x: sum(x) / len(x))

Sorting tuples by average (Midterm Q8)

# x = one tuple like (2, 9, 4)
# property = average = sum(x) / len(x)
sorted(tuples_list, key=lambda x: sum(x) / len(x))

More key= patterns you must know

# Sort strings by LAST character
sorted(["hello", "world", "abc"], key=lambda s: s[-1])

# Sort by second element of tuple
sorted([(1,5), (3,1), (2,8)], key=lambda t: t[1])

# Sort by absolute value (descending) — use reverse=True
sorted([-5, 3, -1, 8], key=lambda x: abs(x), reverse=True)

# Sort dicts by a key
students = [{"name": "Ali", "gpa": 3.5}, {"name": "Bo", "gpa": 3.9}]
sorted(students, key=lambda d: d["gpa"])

3. List Comprehension — Deep Dive

This is the most important section. The midterm constantly asks you to condense logic into a single line. You must be able to read and write these instantly.

The Anatomy (4 Levels of Difficulty)

Level 1: Basic Filter

# Template:  [value  for item in iterable  if condition]
# English:   "Give me every item from the list WHERE condition is true"

# Example: Even numbers from 1 to 10
[x for x in range(1, 11) if x % 2 == 0]
# → [2, 4, 6, 8, 10]

Level 2: Transform + Filter

# Template:  [transform(item)  for item in iterable  if condition]
# English:   "Give me the TRANSFORMED version of each item WHERE condition"

# Example: Squares of odd numbers only
[x**2 for x in range(1, 11) if x % 2 != 0]
# → [1, 9, 25, 49, 81]

Level 3: Ternary (if-else) Inside the Expression

⚠️ WATCH OUT: When if-else is in the expression part (before for), it's a ternary — it runs on EVERY item. When if is after the for, it's a filter.

# Template:  [A if condition else B   for item in iterable]
# English:   "For EVERY item, give me A if true, otherwise B"

# Example: Replace negatives with 0
nums = [3, -1, 5, -7, 2]
[x if x > 0 else 0 for x in nums]
# → [3, 0, 5, 0, 2]

# COMBINING both ternary AND filter:
# [A if cond1 else B  for x in iterable  if cond2]
["big" if x > 5 else "small" for x in range(1,11) if x % 2 == 0]
# → ["small", "small", "big", "big", "big"]
# Only even numbers pass the filter, then each gets labeled

Level 4: Nested Comprehension (Comprehension INSIDE Comprehension)

This is the hardest pattern — and it's exactly what makes the Perfect Numbers question (Q5) so tricky. 

# Template:
# [outer_expr  for outer in range(...)  if outer_cond(INNER_COMP)]
#
# where INNER_COMP is itself a comprehension or generator:
#   sum(inner for inner in range(...) if inner_cond)

# The key insight: The INNER part produces a value
# (like a sum), and the OUTER part uses that value
# to decide whether to keep the item.

🧠 How to Build a Nested One-Liner (Step by Step)

When you see a hard question, don't try to write it all at once. Build it in layers:

  1. Step 1 — The Outer Shell: What are you looping over? Write [x for x in range(...)]
  2. Step 2 — The Condition: What must be true? Add if x == something
  3. Step 3 — The "Something": If "something" requires its own loop, write a generator expression: sum(i for i in range(...) if ...)
  4. Step 4 — Plug it in: Replace "something" with the generator from Step 3.

Worked Example: Perfect Numbers (Midterm Q5)

# GOAL: All numbers 1..n where number == sum of its divisors

# Step 1 — Outer shell: check every number
# [x for x in range(1, n+1)]

# Step 2 — Condition: x must equal sum of divisors
# [x for x in range(1, n+1) if x == sum_of_divisors]

# Step 3 — sum_of_divisors needs its own loop:
# sum(i for i in range(1, x) if x % i == 0)
# This loops 1 to x-1, keeps only numbers that divide x evenly

# Step 4 — Plug it all together:
[x for x in range(1, n+1) if x == sum(i for i in range(1, x) if x % i == 0)]

# For n=30 → [6, 28]

Worked Example: Primes (Classic Exam Question)

# GOAL: All prime numbers from 2 to n

# A prime has NO divisors between 2 and itself.
# Step 1: [x for x in range(2, n+1)]
# Step 2: if x is prime — meaning no number 2..x-1 divides it
# Step 3: "no number divides it" = all(x % i != 0 for i in range(2, x))

[x for x in range(2, n+1) if all(x % i != 0 for i in range(2, x))]

# For n=20 → [2, 3, 5, 7, 11, 13, 17, 19]

4. Nested Comprehensions — Flattening & 2D Loops

Sometimes the midterm gives you a 2D list and asks you to flatten it or do a double loop. The reading order is: left to right = outer to inner.

Flattening a 2D List

matrix = [[1, 2, 3], [4, 5], [6, 7, 8, 9]]

# Multi-line equivalent:
# result = []
# for row in matrix:        ← OUTER loop (written first)
#     for val in row:        ← INNER loop (written second)
#         result.append(val)

# One-liner:
[val for row in matrix for val in row]
# → [1, 2, 3, 4, 5, 6, 7, 8, 9]

⚠️ Reading Order Rule: In a nested comprehension, the for loops are written in the same order as you would write them normally (outer first, inner second). The expression goes at the very front.

2D List with Condition

# Get all even numbers from a 2D list
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

[val for row in matrix for val in row if val % 2 == 0]
# → [2, 4, 6, 8]

Building a 2D List (List of Lists)

# Create a 3x3 multiplication table
[[i * j for j in range(1, 4)] for i in range(1, 4)]
# → [[1, 2, 3], [2, 4, 6], [3, 6, 9]]

# Here the OUTER comprehension makes rows,
# and the INNER comprehension makes columns.

5. Generator Expressions Inside sum(), all(), any()

This is the secret weapon for midterm one-liners. You can put a generator expression (looks just like a comprehension but without square brackets) directly inside built-in functions.

Function What it does Example
sum(...) Adds up all values sum(x for x in range(5)) → 10
all(...) True if ALL are True all(x > 0 for x in [1,2,3]) → True
any(...) True if ANY is True any(x > 5 for x in [1,2,9]) → True
max(...) Largest value max(len(w) for w in ["hi","hello"]) → 5
min(...) Smallest value min(x**2 for x in [-3,1,2]) → 1 
# WHY THIS MATTERS:
# When the midterm says "return if a number's divisors sum to itself"
# you NEED sum() with a generator inside it:

sum(i for i in range(1, x) if x % i == 0)
#    ↑ generator expression — no brackets needed inside sum()

# When it says "check if a number is prime"
# you NEED all() with a generator:

all(x % i != 0 for i in range(2, x))
# "True if x is NOT divisible by ANY number from 2 to x-1"

6. The Late Binding Gotcha (Q2)

When lambdas are created inside a loop, they do NOT capture the value of the loop variable. They only remember the variable's NAME and look it up later.

# ❌ THE TRAP:
p = [lambda: i for i in range(5)]
# By the time the loop ends, i = 4
print([f() for f in p])
# OUTPUT: [4, 4, 4, 4, 4]

# ✅ THE FIX: Default argument captures value at creation time
p = [lambda j=i: j for i in range(5)]
print([f() for f in p])
# OUTPUT: [0, 1, 2, 3, 4]

Rule: Body variables → looked up at call time. Default arguments → evaluated at creation time.

7. 🏟️ Practice Arena — 12 Problems

Try each one on paper first, then reveal the answer. Problems are tagged by difficulty.

Problem 1: Sort by Length Easy

Sort words = ["apple", "bat", "banana", "cat"] by length (shortest first). One line.

sorted(words, key=lambda x: len(x))
# → ['bat', 'cat', 'apple', 'banana']
Problem 2: Sort Tuples by Sum Easy

Sort points = [(2,9), (1,15), (5,4)] by the sum of coordinates.

sorted(points, key=lambda t: sum(t))
# → [(5,4), (2,9), (1,15)]   (sums: 9, 11, 16)
Problem 3: Squares of Evens Easy

Write a one-liner that returns the squares of all even numbers from 1 to 15.

[x**2 for x in range(1, 16) if x % 2 == 0]
# → [4, 16, 36, 64, 100, 144, 196]
Problem 4: Flatten a 2D List Medium

Given matrix = [[1,2],[3,4],[5,6]], write a one-liner that flattens it to [1,2,3,4,5,6].

[val for row in matrix for val in row]
# → [1, 2, 3, 4, 5, 6]
# Reading order: outer for first, inner for second
Problem 5: Late Binding — What's the Output? Medium
funcs = [lambda: x * 10 for x in [1, 2, 3]]
print([f() for f in funcs])

Output: [30, 30, 30]

No default arg → all lambdas see final value of x which is 3. So 3 * 10 = 30 for all.

Problem 6: Late Binding — Fixed Version Medium
funcs = [lambda val=x: val * 10 for x in [1, 2, 3]]
print([f() for f in funcs])

Output: [10, 20, 30]

Default arg val=x captures value at creation time. Each lambda locks in 1, 2, 3.

Problem 7: Sort Strings by Last Character Medium

Sort names = ["Alice", "Bob", "Charlie", "Dave"] by the last letter of each name.

sorted(names, key=lambda s: s[-1])
# → ['Bob', 'Charlie', 'Dave', 'Alice']
# Last chars: b, e, e, e → sorted: b, e, e, e
# (Python's sort is stable, so ties keep original order)
Problem 8: One-Liner Primes Hard

Write a one-liner function func_primes(n) that returns all prime numbers from 2 to n. Body must be one line. No helper functions.

def func_primes(n):
    return [x for x in range(2, n+1) if all(x % i != 0 for i in range(2, x))]

# func_primes(20) → [2, 3, 5, 7, 11, 13, 17, 19]

# Breakdown:
# Outer: x goes from 2 to n
# Condition: all(x % i != 0 ...) — TRUE only if NO number divides x
# Inner generator: checks divisibility for i from 2 to x-1
Problem 9: Flatten + Filter Hard

Given data = [[1,-2,3],[-4,5],[6,-7,8,-9]], write a one-liner that returns all positive numbers from the entire 2D list.

[val for row in data for val in row if val > 0]
# → [1, 3, 5, 6, 8]

# outer loop: each row (sub-list)
# inner loop: each val in that row
# filter: only keep positives
Problem 10: Custom Zip One-Liner Hard

Given data = [[1,2,3],[4,5,6],[7,8]], write a one-liner that zips them (truncating to shortest). Expected: [(1,4,7),(2,5,8)]. Hint: use min() and len().

data = [[1,2,3],[4,5,6],[7,8]]

[tuple(lst[i] for lst in data) for i in range(min(len(lst) for lst in data))]
# → [(1, 4, 7), (2, 5, 8)]

# Breakdown:
# min(len(lst) for lst in data) → finds shortest list length (2)
# for i in range(2) → i = 0, then i = 1
# tuple(lst[i] for lst in data) → grabs index i from EVERY sub-list
Problem 11: Max Difference Tuple Hard

Write a lambda max_diff that takes a list of tuples (each with 2 ints) and returns the tuple with the greatest absolute difference. One line, no helpers.

max_diff = lambda lst: max(lst, key=lambda t: abs(t[0] - t[1]))

# max_diff([(1,2), (5,9), (10,2)]) → (10, 2)
# abs(10-2) = 8, which is the biggest difference
Problem 12: Perfect Numbers (Midterm Q5 Exact) Hard

Write func_perfect(n) — returns all perfect numbers from 1 to n. Body is one line. A perfect number equals the sum of its proper divisors (excluding itself).

def func_perfect(n):
    return [x for x in range(1, n+1) if x == sum(i for i in range(1, x) if x % i == 0)]

# func_perfect(30) → [6, 28]

# Build process:
# 1. Outer: [x for x in range(1, n+1) ...]      → check each number
# 2. Condition: if x == sum(...)                  → equals sum of divisors?
# 3. Inner: sum(i for i in range(1,x) if x%i==0) → add up all divisors of x
